Changelog

Version 1.1.2 - 10/30/2024

This release has several bug fixes and small UI updates

  • Fixes for stability issues caused by concurrent UI updates.
  • Hide or disable UI that is not relevant for profiling.
  • Make "open function" mouse and keyboard shortcuts consistent across all views, see updated documentation.
  • Update syntax highlighting definitions for x64 and ARM64.
  • Fix N^2 step in building the Call Tree view.
Version 1.1.0 - 10/09/2024

First public release of Profile Explorer.

Version 1.0.9 - 08/07/2024

Main additions and improvements

  • Downloading of binaries and PDBs from the new Cloud Symweb is now supported, being the default private symbol server.
  • Significant speedup of large trace loading by multi-threading the rest of the processing pipeline. This also applies to filtering based on threads/time range using the Timeline panel.
  • General speedup of loading and switching functions in Assembly view and Source file panel by tuning profile processing and UI, reducing GC pressure, more caching, etc.

Bug fixes and other improvements

  • Fix for very slow trace loading seen with certain traces.
  • Fixed total time computation for recursive functions displayed in Functions/Modules tab of Details panel.
  • Fixed selection of functions in the Flame graph from other panels (done through right-click context menu).
  • Fix initial jump to hottest line in preview popup and color marking on scroll bar.
  • Respect C++ demangling settings from the Summary panel options.
  • Multiple other smaller bug fixes, UI tweaks and polish.
  • Redesigned Start page to include a list of previous profiling sessions.
  • Simplified function/module marking settings by having them into one place under "Profile Marking" in the Settings window and the new wrench icon in the main app menu.
  • Removed Modules/Functions buttons from the Summary/Flame Graph/Call Tree panels since they were redundant with the ones in the main app menu.
  • Option to disable animations in the UI (current applies to the Flame graph panel). If the app runs through a Remote Desktop connection this is enabled automatically.
  • Option to limit number of CPU cores used during trace loading and other profiling UI tasks, by default 75% of logical cores are used.
Version 1.0.8 - 06/14/2024
  • Multiple bug fixes, general stability improvement and more UI polishing.
  • Added a symbol cache system to save the result of processing PDB files. This significantly speeds up future loading of traces using the same PDB, especially for apps with large PDBs like Edge (5.7 GB): 1 min 13 sec for first trace load to 14 sec afterwards
  • Speed up of initial trace loading with large PDBs by optimizing DIA access and other tuning. For an Edge trace, loading time drops to 1 min 13 sec from the previous 2 min 8 sec.
  • Much quicker opening and switching of function assembly view and source file with large PDBs.
  • Improved UI for managing symbol paths in the Load Profile window: list entries are path auto-complete boxes, option to select path using an open file dialog.
Version 1.0.7 - 05/22/2024

Main additions and improvements

  • Filtering by instance and thread in assembly view, source code and preview popup.
  • Combined source code + assembly sections in Source File panel.
  • Source code parsing and statement profile time marking (loops, if/else, switch)
  • Inlinees (inlined functions) summary menu in assembly view.
  • Function and Module marking, synchronized between panels, saved between sessions.
  • Profile Overview with predefined categories like "Memory allocation".
  • Threads section and Categories tab in Flame Graph Details panel.
  • Open function by double-clicking call target name in assembly view and preview popup.
  • Back/next history buttons in assembly view and preview popup.
  • Copy and Exporting as HTML/Markdown in all profiling panels.

Bug fixes, performance, and other improvements

  • Support for circular ETW traces, often collected by WPR and telemetry.
  • Trace loading speedup for large ETL files (up to 3x), faster symbol loading.
  • Lots of bug fixes (general app stability, UI polish, Flame Graph text clarity, removing flickering in assembly view).
  • Faster assembly/source code switching, timeline selection, inter-panel synchronizing, reduced UI delays.
  • Rust support, .NET profiling when starting a new app with JIT assembly capture.
  • GraphViz binaries are included, required for viewing Flow Graphs.
  • Saving of width and order of list view columns in profiling panels.
  • Wildcard search in Summary, Call Tree, and Flame Graph panels.
  • Whole app UI scaling between 50% and 200%, with 5% increments.
  • Horizontal scrolling with touchpad in all list views and Flame Graph.
  • Improved tooltips for assembly view and preview popup.
